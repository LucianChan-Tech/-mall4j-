## 一、前置环境要求

| 工具 | 最低版本 | 检查命令 | 说明 |
|------|---------|----------|------|
| JDK | 17+ | `java -version` | 推荐使用 JDK 21 |
| Maven | 3.6+ | `mvn -version` | 项目管理与构建 |
| Node.js | >=18 | `node -v` | 前端运行环境 |
| npm | >=9 | `npm -v` | 前端依赖管理 |
| pnpm | >=6 | `pnpm -v` | mall4v 前端依赖管理 |
| Docker | 任意 | `docker --version` | 用于运行 Redis 等中间件 |
| MySQL | 8.x | `net stat -ano \| findstr :3306` | 数据库服务必须运行 |

## 二、启动流程概览

启动顺序（**依次执行，不可颠倒**）：

```
第1步 ──→ MySQL（确保数据库服务运行 + 已导入数据）
第2步 ──→ Redis（Docker 启动）
第3步 ──→ 编译后端（Maven 打包）
第4步 ──→ 启动 admin 后端（端口 8085）
第5步 ──→ 启动 api 后端（端口 8086）
第6步 ──→ 启动 mall4v 后台管理前端（端口 9527）
第7步 ──→ 启动 mall4uni 商城前端（端口 5173）
```

> **前端开发时可直接用 Vite Dev Server，不依赖 Docker 部署前端。**

## 三、基础设施启动

### 3.1 MySQL 数据库

**方式 A：本地 MySQL 服务**

1. 确保 MySQL 8.x 已安装并启动：
   ```bash
   # 检查 MySQL 服务是否运行
   netstat -ano | findstr :3306
   ```

2. 创建数据库并导入表结构：
   ```bash
   # 登录 MySQL（按提示输入密码）
   mysql -u root -p

   # 创建数据库（如果尚未创建）
   CREATE DATABASE IF NOT EXISTS `yami_shops` DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_general_ci;

   # 退出 MySQL
   exit

   # 导入数据
   mysql -u root -p yami_shops < db/yami_shop.sql
   ```

3. 确认后端配置中的密码与本地 MySQL 密码一致（见 `yami-shop-admin` 和 `yami-shop-api` 的 `application-dev.yml`）。

**方式 B：Docker MySQL（可选）**

```bash
docker compose up -d mall4j-mysql
```

> 首次启动会自动执行 `db/yami_shop.sql` 初始化数据库。

---

### 3.2 Redis（Docker）

```bash
# 如果已有同名容器，先删除
docker rm -f yami-redis 2>/dev/null

# 启动 Redis 5.0.4
docker run -d \
  --name yami-redis \
  -p 6379:6379 \
  redis:5.0.4

# 验证是否启动成功
docker ps | grep yami-redis
```

## 四、后端启动

### 4.1 编译后端

在项目根目录执行：

```bash
# 编译全部后端模块（跳过测试，加快速度）
mvn clean package -DskipTests
```

> 如果网络慢，Maven 已配置阿里云镜像仓库（见 `pom.xml`）。

**只编译特定模块：**

```bash
# 只编译 admin 模块及其依赖
mvn clean package -pl yami-shop-admin -am -DskipTests

# 只编译 api 模块及其依赖
mvn clean package -pl yami-shop-api -am -DskipTests
```

---

### 4.2 启动 admin 后端（后台管理 API）

```bash
java -jar -Dspring.profiles.active=dev -Xms512m -Xmx512m ^
  yami-shop-admin/target/yami-shop-admin-0.0.1-SNAPSHOT.jar
```

- 端口：**8085**
- Profile：`dev`（开发环境，连接本地 MySQL + 本地 Redis）
- 启动成功标志：日志中出现 `Tomcat started on port 8085`

---

### 4.3 启动 api 后端（商城 API）

> 先确保 admin 后端启动完成。

```bash
java -jar -Dspring.profiles.active=dev -Xms512m -Xmx512m ^
  yami-shop-api/target/yami-shop-api-0.0.1-SNAPSHOT.jar
```

- 端口：**8086**
- Profile：`dev`
- 启动成功标志：日志中出现 `Tomcat started on port 8086`

---

### 4.4 重新编译并重启后端（修改代码后）

```bash
# 停止旧的 Java 进程（Ctrl+C 或关闭终端）

# admin 重新编译并启动
mvn clean package -pl yami-shop-admin -am -DskipTests
java -jar -Dspring.profiles.active=dev -Xms512m -Xmx512m ^
  yami-shop-admin/target/yami-shop-admin-0.0.1-SNAPSHOT.jar

# api 重新编译并启动（另一个终端）
mvn clean package -pl yami-shop-api -am -DskipTests
java -jar -Dspring.profiles.active=dev -Xms512m -Xmx512m ^
  yami-shop-api/target/yami-shop-api-0.0.1-SNAPSHOT.jar
```

## 五、前端启动

### 5.1 mall4v —— 后台管理前端

| 属性 | 说明 |
|------|------|
| 目录 | `front-end/mall4v/` |
| 技术栈 | Vue 3 + Vite + Element Plus + Avue |
| 包管理器 | **pnpm** |
| 开发端口 | 9527 |
| API 代理 | 请求转发到 `http://127.0.0.1:8085`（admin 后端） |

**启动命令：**

```bash
# 安装依赖（首次启动或依赖有变更时执行）
pnpm --prefix front-end/mall4v install

# 启动开发服务器
pnpm --prefix front-end/mall4v dev
```

> 自动打开浏览器 `http://localhost:9527`

**生产构建（可选）：**

```bash
pnpm --prefix front-end/mall4v build
# 构建产物在 front-end/mall4v/dist/
```

---

### 5.2 mall4uni —— 商城 H5 前端

| 属性 | 说明 |
|------|------|
| 目录 | `front-end/mall4uni/` |
| 技术栈 | Vue 3 + Vite + Tailwind CSS + GSAP + Pinia |
| 包管理器 | **npm** |
| 开发端口 | 5173 |
| API 代理 | 请求转发到 `http://127.0.0.1:8086`（api 后端） |

> mall4uni 已从 UniApp 重写为纯 Vue 3 SPA，使用标准 Vite，不再需要 pnpm 或 UniApp CLI。

**启动命令：**

```bash
# 安装依赖（首次启动或依赖有变更时执行）
npm --prefix front-end/mall4uni install

# 启动开发服务器
npm --prefix front-end/mall4uni run dev
```

> 访问 `http://localhost:5173`

**生产构建（可选）：**

```bash
npm --prefix front-end/mall4uni run build
# 构建产物在 front-end/mall4uni/dist/
```

## 七、常见问题速查

### Q1：MySQL 连接失败 —— `Access denied for user 'root'@'localhost'`

**原因**：`application-dev.yml` 中密码与本地 MySQL 实际密码不匹配。

**解决**：修改两个后端模块的配置：
- `yami-shop-admin/src/main/resources/application-dev.yml`
- `yami-shop-api/src/main/resources/application-dev.yml`

```yaml
spring:
  datasource:
    password: '你的MySQL密码'   # 如果密码含特殊字符如 #，必须加引号
```

修改后需重新编译对应模块并重启。

---

### Q2：Maven 编译失败 —— `Failed to delete xxx.jar`

**原因**：旧的 `java -jar` 进程还在运行，占用了 JAR 文件。

**解决**：
```bash
# 停止所有 Java 进程（谨慎使用）
taskkill /F /IM java.exe
# 或手动关闭后台终端
```

---

### Q3：滑块验证码失败

**原因**：Redis 未运行，滑块验证码依赖 Redis 存储验证状态。

**解决**：确认 `docker ps` 中有 Redis 容器在运行。

---

### Q4：mall4v 前端启动报 `pnpm: command not found`

**原因**：未安装 pnpm。

**解决**：
```bash
npm install -g pnpm
```

---

### Q5：mall4uni 页面空白/白屏

**原因**：JS 运行时错误或依赖未安装。

**解决**：
```bash
# 检查控制台错误（F12 → Console）
# 确保依赖已安装
npm --prefix front-end/mall4uni install
```

```bash
java -jar -Dspring.profiles.active=dev -Xms512m -Xmx512m ^

```
