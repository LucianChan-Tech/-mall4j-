## 四、完整启动步骤

> ⏱ 总耗时约 2-3 分钟（首次需额外 5-10 分钟下载依赖）

### 第 1 步：检查基础设施

```bash
# 1.1 检查 MySQL 是否运行
netstat -ano | findstr :3306
# 应看到 LISTENING 状态

# 1.2 检查 Docker 是否登录（拉取镜像需要）
docker login
# 如果凭证过期，手动在 Docker Desktop 中重新登录
```

### 第 2 步：启动 Redis（Docker）

```bash
# 如果容器已存在，先删除再重新运行
docker rm -f yami-redis 2>/dev/null
docker run -d --name yami-redis -p 6379:6379 redis:5.0.4
```

> ⚠️ 如果 Docker 拉取镜像失败（网络不通），检查 Docker Desktop 是否登录，或尝试重启 Docker Desktop。

### 第 3 步：编译后端

```bash
# 编译全部后端模块（跳过测试）
mvn clean package -DskipTests
```

> **如果只需要编译某个模块：**
> ```bash
> mvn clean package -pl yami-shop-admin -am -DskipTests  # 仅 admin
> mvn clean package -pl yami-shop-api -am -DskipTests    # 仅 api
> ```
>
> ⚠️ 如果编译失败提示 `Failed to delete xxx.jar`，说明有旧的 Java 进程占用 JAR 文件，先停掉旧进程再编译。

### 第 4 步：启动后端服务

```bash
# 启动 admin 后台管理 API（端口 8085）
java -jar -Dspring.profiles.active=dev -Xms512m -Xmx512m ^
  yami-shop-admin/target/yami-shop-admin-0.0.1-SNAPSHOT.jar

# 启动 api 商城 API（端口 8086）
java -jar -Dspring.profiles.active=dev -Xms512m -Xmx512m ^
  yami-shop-api/target/yami-shop-api-0.0.1-SNAPSHOT.jar
```

> ✅ 启动成功的标志：日志最后出现 `Tomcat started on port 8085/8086`，无 MySQL 报错。

### 第 5 步：启动 mall4v 后台管理前端

**方式 A：Vite Dev Server（推荐开发调试）**

```bash
pnpm --prefix front-end/mall4v dev
# 运行在 http://localhost:9527
```

**方式 B：Docker Nginx（生产模式）**

```bash
cd front-end/mall4v
pnpm install
pnpm build
docker build -t mall4v-frontend .
docker run -d --name mall4v-frontend -p 8080:80 mall4v-frontend
```

### 第 6 步：启动 mall4uni 商城前端

```bash
# 6.1 安装依赖
pnpm --prefix front-end/mall4uni install

# 如果 pnpm 报错 "ERR_PNPM_IGNORED_BUILDS"，需要修改 pnpm-workspace.yaml：
# 将 allowBuilds 下的所有项改为 true:
#   allowBuilds:
#     core-js: true
#     esbuild: true
# 然后重新执行 pnpm install

# 6.2 启动 H5 开发服务器
pnpm --prefix front-end/mall4uni dev:h5
# 运行在 http://localhost:5174
```

## 六、问题排查清单

### 问题 1：MySQL 连不上——`Access denied for user 'root'@'localhost'`

**现象**：后端启动日志显示 `Access denied for user 'root'@'localhost' (using password: YES)`

**原因**：`application-dev.yml` 中的 `password` 与本地 MySQL 实际密码不匹配

**解决**：
```yaml
# yami-shop-admin/src/main/resources/application-dev.yml
# yami-shop-api/src/main/resources/application-dev.yml
spring:
  datasource:
    password: '#Cwy050525'   # 注意 # 号需要引号包裹
```

> 修改后需要重新编译对应的后端模块并重启。

---

### 问题 2：MySQL 间歇性报错——`Public Key Retrieval is not allowed`

**现象**：后端偶发 500 错误，日志显示 `Public Key Retrieval is not allowed`

**原因**：MySQL 8.x 默认使用 `caching_sha2_password` 认证插件，JDBC 驱动需要额外参数

**解决**：在 JDBC URL 末尾添加 `&allowPublicKeyRetrieval=true`

```yaml
url: jdbc:mysql://127.0.0.1:3306/yami_shops?allowMultiQueries=true&useSSL=false&useUnicode=true&characterEncoding=UTF-8&autoReconnect=true&zeroDateTimeBehavior=convertToNull&useJDBCCompliantTimezoneShift=true&useLegacyDatetimeCode=false&serverTimezone=GMT%2B8&nullCatalogMeansCurrent=true&allowPublicKeyRetrieval=true
```

> 修改后需要重新编译对应的后端模块并重启。

---

### 问题 3：Maven 编译失败——`Failed to delete xxx.jar`

**现象**：`mvn clean package` 失败，提示 `Failed to clean project: Failed to delete xxx.jar`

**原因**：旧的 `java -jar` 进程正在运行，占用了 JAR 文件

**解决**：先停止所有 Java 后端进程，再编译：
```bash
# 停止所有 Java 进程（谨慎使用）
taskkill /F /IM java.exe
# 或手动关闭对应的终端窗口
```

### 问题 4：前端 Docker 显示 "Welcome to Nginx"

**现象**：访问 `localhost:8080` 显示 Nginx 默认欢迎页

**原因**：
1. `nginx.conf` 中 `try_files` 的 fallback 路径是 `/` 而不是 `/index.html`
2. Nginx 基础镜像自带的 `default.conf` 冲突

**解决**：确保 `nginx.conf` 配置正确：

```nginx
server {
    listen 80 default_server;
    server_name localhost;
    root /usr/share/nginx/html/dist;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

Dockerfile 中删除默认配置：

```dockerfile
RUN rm -f /etc/nginx/conf.d/default.conf
COPY ./nginx.conf /etc/nginx/conf.d/default.conf
```

### 问题 5：前端请求 API 报错——`host.docker.internal` 不识别

**现象**：前端页面能加载，但 API 请求失败，控制台报网络错误

**原因**：`.env.production` 中用了 `VITE_APP_BASE_API = 'http://host.docker.internal:8085'`，`host.docker.internal` 只在 Docker 容器内部有效，**Windows 浏览器不认识这个地址**

**解决**：始终使用 `127.0.0.1` 或 `localhost`：

```
VITE_APP_BASE_API = 'http://127.0.0.1:8085'   # mall4v
VITE_APP_BASE_API = 'http://127.0.0.1:8086'   # mall4uni
```

> 修改后需要重新 `pnpm build` 并重建 Docker 镜像。

### 问题 6：pnpm 安装报错——`ERR_PNPM_IGNORED_BUILDS`

**现象**：`pnpm install` 成功但 `pnpm dev` 时失败，提示 `Ignored build scripts`

**原因**：pnpm 11 默认不执行某些包的构建脚本，需要显式批准

**解决**：修改 `front-end/mall4uni/pnpm-workspace.yaml`（或 `mall4v` 的）：

```yaml
allowBuilds:
  core-js: true
  esbuild: true
```

然后重新执行 `pnpm install`。

### 问题 9：UniApp 端口配置不生效

**现象**：修改了 `vite.config.js` 中的 `port`，但启动后端口没变

**原因**：UniApp 框架从 `src/manifest.json` 中读取 `devServer.port`，而不是 `vite.config.js`

**解决**：需要同时修改两处：
1. `front-end/mall4uni/vite.config.js` → `server.port`
2. `front-end/mall4uni/src/manifest.json` → `devServer.port`

### 问题 11：AI 购物助手返回模拟回答而非真实 AI 回答

**现象**：AI 助手的回答很模板化（如 `本店满99元包邮哦～`），没有智能感

**原因**：`ChatService` 中检测到未配置有效的 DeepSeek API Key，自动降级为关键词匹配的模拟回答

**解决**：检查 `yami-shop-api/src/main/resources/application.yml` 中的配置：
```yaml
deepseek:
  api-key: sk-xxx   # 需要在 https://platform.deepseek.com/ 注册获取
  api-url: https://api.deepseek.com
```
- 如果 `api-key` 为空或等于 `sk-your-key-here`，则走模拟回答
- 配置有效的 API Key 后重新编译并重启即可接入真实 AI
- 无需 API Key 时模拟回答也能正常使用，覆盖常见问题

```bash
# 快速测试 AI 助手（不需 API Key 也能返回模拟回答）
curl -s http://127.0.0.1:8086/chat/send -X POST -H "Content-Type: application/json" -d "{\"question\":\"包邮吗？\",\"productId\":null}"

# 带商品上下文的提问（传入商品ID）
curl -s http://127.0.0.1:8086/chat/send -X POST -H "Content-Type: application/json" -d "{\"question\":\"这个商品多少钱？\",\"productId\":74}"
```

正确响应示例：
```json
{"code":"00000","data":"这个商品的价格是¥100.0。不过目前库存为0，暂时无法购买哦。","success":true}
```

### 7.5 查看服务状态

```bash
docker ps -a                        # Docker 容器状态
netstat -ano | findstr :8085        # 检查 admin 端口
netstat -ano | findstr :8086        # 检查 api 端口
netstat -ano | findstr :3306        # 检查 MySQL 端口

```
