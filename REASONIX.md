# Reasonix Code 工作规则

## 问清再跑 (Ask-Before-Run)

**每次跑任务时，有疑问的地方直接问我，不要运行。** 一有疑问就问我，直到你对任务的过程以及想要得到的结果完全了解之后再跑任务。

### 触发场景

- 任务描述模糊、不完整 → 先问
- 涉及多个可能的实现方式，不确定选哪个 → 先问
- 影响范围不清晰（改哪些文件、改到什么程度）→ 先问
- 要跑的命令可能影响环境或数据 → 先问
- 任何你不太确定的地方 → **先问，不问就默认继续运行**

### 为什么

避免猜测意图后做了无用功，浪费 token 和时间。问清楚再动手，反而更快。
